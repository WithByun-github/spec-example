# 04 — 한국형 PII 마스킹 (입·출력 양방향)

Status: ready-for-agent

> 출처: `docs/PRD.md` (F1-4, F5, 성공기준 ③), `docs/TASKS.md` (Phase 3). 03과 병렬 진행 가능.

## What to build

한국형 개인정보를 입력단(LLM 도달 전)과 출력단(사용자 노출 전) 양방향에서 탐지·마스킹한다. Presidio 글로벌 엔진에 한국 특화 패턴을 더하고, 체크섬 검증으로 오탐을 줄인다. 사내 데이터가 외부 SaaS LLM으로 새어 나가는 것을 막는 핵심 기능이다.

## Acceptance criteria

- [ ] `app/guards/pii_ko.py` — Presidio + 한국형 패턴(주민등록번호·휴대폰·카드·사업자등록번호·계좌·여권)
- [ ] 체크섬 검증으로 오탐 감소 (예: 주민번호 체크섬)
- [ ] 입력단 마스킹(LLM 도달 전) + 출력단 마스킹(사용자 노출 전), gateway 입·출력에 연결
- [ ] **검증**: 주민번호 포함 입력 → 마스킹 후 LLM 전달 / 응답에 주민번호 → 사용자 노출 전 마스킹
- [ ] 마스킹된 형태만 로그/저장에 남기고 평문은 저장하지 않음
- [ ] 데모 데이터는 합성값만 사용 (실제 개인정보 금지)
